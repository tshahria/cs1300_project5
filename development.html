Reacting Positively to ReactJS

Overview

I decided to create this simple webapp to learn how to use ReactJS. As someone who is into fashion, I obviously decided to create a website related to that for my first webapp.

Goal and Value

The goal of the application is to allow the user to purchase clothes while filtering by region it was made, and the size of the clothing. The user can also sort by name or by price.

The user can add to cart which will show on the right side of the screen, along with quantity of that item, price of that item and the total price. The user can also choose to add/remove by either clicking on the button on the item itself, or he/she can just remove all of the specific item from the cart by clicking the trashcan logo in the cart.

Usability Principles

I will break down my application in terms of UIUX usability principles

  • Learnability
    • The website is very easy to learn as its very minimalistic and lacks a lot of clutter. Thus the user should be able to easily understand how to use it.
      • The trash can logo on the cart item implies removing it from the cart
      • The plus/minus buttons means adding to the cart
      • There is a breakdown of the total of a certain item based on it’s price and quantity, as well as a total of the entire cart.
      • The methods by which the user can filter and sort are at the very top, and the naming is very intuitive.
    Memorability
    • The user will be able to use the website with ease after coming back after some time, as the design/usage isn’t complicated and very intuitive. The same points I mentioned above in making my website learnable also makes it very memorable.
    Usability
    • Overall, I think the Learnability and Memorability of the website is very low. Thus, I don't think the website is really usable due to the flaws I outlined above
