Overview

A/B Testing is something used commonly in the real world to test small changes among a group of users. To learn more about A/B testing and how it is used in the real world, I decided to test my changes on a website with regards to Time To Completion and Return Rate. Metrics

I plan to look at two metrics for the changes between my websites: Time to Completion, and Return Rate. Below I will outline the Null and Alternative Hypothesis for these metrics.

  • Time to Completion
    • Null Hypothesis: There will be no significant difference between the time to completion of both pages.
    • Alternative: The time to completion will be different due to the change in layout and button size between the two pages. The 2x2 layout with smaller buttons (Version A) should result in a lower time to completion than the large add to cart buttons as users have to scroll less to find their items.
    Return Rate
    • Null Hypothesis: There will be no significant difference between the time to return rate of both pages.
    • Alternative: The return rate will be different due to the addition of a total to one of the pages (to version B), which will mean the user will have less need to return to the main page

    • Calculations and Conclusion

    This is the calculation based on the data

  • Time to Completion (T Test)
    • The P Value was 0.434 and the t score was -0.169. As P>0.05, we can’t reject the null, and the results are statistically insignificant, so we don’t know if the alternative hypothesis will repeat itself in other trials.
    • Conclusion: No significant difference.
    Return Rate (Chi^2 Test)
    • The p value I got for the return rate was 0.636 after running a chi2 test, and the chi2 value was 0.224. As 0.636>0.05, this means we can’t reject the null. Hence, we can’t reject the null, and the yur results are statistically insignificant, so we don’t know if the alternative hypothesis will repeat itself in other trials./li>
    • Conclusion: No significant difference.

    Takeaways

    • Limitations: I think the biggest limitation for this A/B testing was the quantity and quality of user data. Since most of the users were other students, they were somewhat biased when interacting with the site. We also only had about 20 data points to interpret from, which is very little
    • I think in general, in terms of design principles, I think from my results I understood that things we think might incur massive changes have no effects. However, that isn't always the case. Sometimes even a small change (color, small change in affordance), can lead to massive changes.
